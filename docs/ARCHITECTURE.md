# Архитектура Jarvis

Jarvis построен по модульной архитектуре.

Основные компоненты:

WakeWord Layer
Отвечает за детекцию активационного слова Jarvis.
Использует Porcupine.

STT Layer
Распознаёт речь пользователя.
Работает локально через Vosk.

Command Engine
Анализирует текст.
Находит команды.
Поддерживает несколько команд в одном предложении.

AI Layer
Используется только если включён.
Не мешает системным командам.

TTS Layer
Озвучивает ответы.
Работает локально.

Config Layer
Хранит настройки пользователя.
Находится в AppData.

Все модули изолированы и легко заменяются.
